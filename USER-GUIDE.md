# 伴生灵魂盘（Soul Framework） · 用户使用手册

> 给**不会训练 AI、不想写代码**的你。
> 目标就一句：让你手头的 AI **聊着聊着就越来越懂你**，不用训练、不用配置环境。

---

## 这是什么（一句话）

一个"伴生灵魂盘"。你本来用的 AI 每次开聊都把你当陌生人、聊完就忘。
把这套框架接上后，它会**自己记住你说的话、你的习惯、你的脾气**，下次开聊自动带上——像真人关系一样越处越熟。

**它不是另一个 AI，是给你现有 AI 装的一个"记事本 + 读心中间件"。**

**它也可以叫"伴生灵魂盘"**：一个完全属于你、只装你信息的本地空间。你换任何 AI，把这个外挂递过去，新 AI 翻一眼就秒懂你——这就是"越用越懂"的真相，不靠训练，靠这个外部记忆层。

> **注意**：不要跟其他记忆类 MCP 混淆。其他记忆类 MCP 是另一种工具，本框架叫 **Soul Framework / 伴生灵魂盘**，是独立的、更完整的人格+记忆引擎。

---

## 它解决什么痛点

| 情况 | 普通 AI | 本框架 |
|------|---------|--------|
| 换新会话 | 把你当陌生人 | 自动带上对你的了解 |
| 换模型/换壳 | 又从零开始 | 记忆在框架里，不丢 |
| 想让它懂你 | 得去"微调训练"（要技术/算力/数据，99%人搞不了） | **直接聊就行**，它自己记 |

## 三步上手

### 第 1 步：填一个配置文件
打开框架里的 `config.schema.json`，把这几项填上（出厂是空的，你自己定）：
- `agent_id`：终身不变的英文标识（比如 `"xiaoming"`），设完锁定——这是多智能体分区的钥匙
- `agent_name`：你的 AI 叫什么（比如"小安"）
- `user_name`：你叫什么（比如"老李"）
- `relationship`：你们是啥关系（比如"工作助手"或"数字伙伴"）
填完保存。

> **多个 AI 共用一个灵魂盘？** 接上 MCP 后依次对每个 AI 说"你的 agent_id 是 XX""你叫 XX"——框架会自动把每个 AI 的独立配置存进 `agents` 映射表，互不覆盖。第一个 AI 设 agent_id 时现有配置自动迁移，之后每新增一个就在对话里设就行。不用手动写 JSON。
>
> **⚠️ 首次接入新 agent 的正确顺序**：① 先调 `get_onboarding(agent_id="XX")` 阅读接入须知；② 调 `get_config` 确认身份；③ 调 `recall_context` 注入上下文；④ 经 `update_config(key="agent_id", value="XX")` 注册。**注册时框架会返回 write_key（明文仅此一次），务必让 AI 妥善保管**——之后改配置、写私密区都要带它，丢了需管理员重置。完整安全模型见 `MCP-接入指南.md` 第二节。

> **嫌手动填麻烦？** 接上 MCP 后，你直接在对话里说"你叫 XX""以后叫我老李""咱是搭档关系"，AI 会经 `update_config` 工具**自己写回配置文件**——不用你碰 JSON。`agent_id` 只可设一次（设完锁定），`agent_name`/称呼/关系/基调随时可改。白名单限定它改不了框架内部路径，安全。

### 第 2 步：选一种方式接上（看你用的 AI 是哪种）
**好消息**：2026 年主流 AI 基本都支持 MCP（WorkBuddy、Claude、Cursor、Cline、OpenClaw 等），所以**首选 B 路 MCP**——挂上就全自动，你只管聊。
只有确认你的 AI 不支持 MCP，才退而用 A 粘贴法兜底。

### 第 3 步：像平常一样开聊
值得记的，它会自己写进 `data/facts/` 文件夹；下次对话自动更懂你。
**你什么都不用做，正常说话就行。**

## 四种接法，按你的实际情况选

> **2026 年现状：MCP 已经是 AI 智能体的"标配"接口。** 主流客户端（WorkBuddy、Claude Desktop、Cursor、Cline、OpenClaw 等）全都支持 MCP。
> 也就是说——**只要你的 AI 支持 MCP（基本都支持），最推荐的就是 B 路：一键挂上，它自己读、自己写，全自动。**
> 只有那种真不支持 MCP 的"老古董"，才退而用 A 粘贴法兜底。

### A. 粘贴法 · 兜底方案（仅给不支持 MCP 的 AI）
你的 AI 万一真不支持 MCP？没关系。框架能生成一段"我了解用户的内容"文本，你**复制，每次开聊先粘进对话框第一段**当开场白即可。
- 怎么生成：跑 `python engine/make_context.py`，文本直接打印在屏幕上，复制。
- 优点：不用装任何东西、不用改代码，**任何 AI 都能用**。
- 缺点：每次开新会话要粘一次（但一键生成，复制粘贴就两下）。
- 适合：极少数不支持 MCP 的闭源 AI（这种现在很少见了）。

### B. MCP 工具 · 推荐绝大多数用户（全自动、实时）
把框架当成一个"MCP 工具"挂到你的 AI 上。挂上后，AI 聊到关键处会**自己调工具去记、去查**，完全不打断对话。
- 优点：全自动、实时、细粒度，最贴近"真人记忆"。**用户完全不操心——你只管正常聊，记和读它自己干。**
- 前提：你的 AI 客户端支持 MCP。**2026 年主流 AI 全支持**（WorkBuddy / Claude / Cursor / Cline / OpenClaw 等）。不支持 MCP 的，基本可以判"太弱"了。
- 数据主权在你：**框架只写自己的 `data/` 本子，记什么、读什么由你的 AI 在对话中自主决定，但最终文件在你本地、你能看能删能改。**
- 怎么接：看 `MCP-接入指南.md`，配几行就完事。

### C. 代码注入 · 仅适合"你能改 AI 的 System Prompt"的开发者 / 自搭智能体
你自己在写 AI、能改它的 system prompt？在对话前把 `Soul().context_pack()` 拼进 system prompt 即可（详见 `integration/README.md` 方式 C）。
- 优点：全自动、无感。
- 适合：开发者、自搭智能体的人。

### D. 文件协议 · 仅适合"有文件权限的 AI / 本地部署"
框架本质就是一个文件夹，里面 `data/facts/*.jsonl` 是纯文本记忆。任何有文件权限的 AI 都能直接读写。复制整个文件夹到任何机器，框架跟着走。
- 优点：零运行时依赖、可审计、极致可移植。
- 适合：想二次开发、或本地跑智能体的人。

> **划重点**：**首选 B（MCP）**——挂一次，全自动，你啥都不用管。A 粘贴法是给不支持 MCP 的老古董兜底的。C / D 是给开发者/自搭用的。

## 怎么看我灵魂盘里存了什么？

接上 MCP 后数据全自动记，但你想亲眼看看存了啥——框架自带**实时仪表盘**：

```bash
python engine/dashboard.py
```

浏览器自动打开 `localhost:8877`，你能看到：

- **全部记忆**：FACT / PREF / BOUND / COMMIT，每条都能搜
- **Vault 走廊 + 房间**：混合记忆层，谁写的、什么时候、什么内容
- **多智能体分区**：公共区 cross（绿色）vs 各智能体私密区（不同颜色），点一下即筛选
- 搜索、排序、分页、自动刷新——数据多了也不卡

**关了浏览器还想看**：再跑一次 `python engine/dashboard.py` 就重新打开，数据永远是最新的。

## 仪表盘登录 & 忘记管理员密码怎么办

仪表盘（`python engine/dashboard.py`）带一道**管理员登录**，防止本机别人随手翻看你的记忆。这里把它怎么工作、密码忘了怎么找回讲清楚。

### 初始密码从哪来

- **首次启动**仪表盘时，框架会自动生成一个随机初始密码，显示在登录页登录框上方（形如 `🔑 初始密码：xxxxxxxxxxxx`）。
- 这个初始密码会**持久化保存**，所以哪怕你关掉仪表盘、重启电脑，只要你**还没主动改过密码**，登录页就会**一直显示**它——不怕没看清就消失。
- 一旦你在仪表盘里**主动改了密码**，登录页就**不再显示**任何初始密码（安全起见），此后用你自己设的新密码登录。

### 忘记密码怎么办（两种方法，推荐第一种）

改过密码又忘了、登录页也不再显示——别慌，本地工具直接重置即可。**记忆数据一条都不会丢。**

#### 方法一（推荐）：一条命令重置

关掉正在运行的仪表盘（终端 `Ctrl+C` 或关掉窗口），在框架目录里运行：

```bash
python engine/dashboard.py --reset-admin
```

它会自动清空旧密码字段、重新生成一个新初始密码并打印在终端（形如 `🔑 新初始密码：xxxxxxxxxxxx`）。**全程不用你手动改任何 JSON，也不会动 `data/` 里的记忆。**

#### 方法二（兜底）：手动删字段

如果暂时没法跑命令，也可以手动编辑 `config.schema.json`：

1. 找到文件末尾自动生成的密码字段（通常在最后几行）。重置**只需删掉这两个密码字段**：
   - `admin_password` —— 当前密码的哈希值
   - `_initial_password_plain` —— 初始密码明文（登录页显示用）

   > `admin_password_changed` 是"是否改过密码"的标记，**删不删都行**，不影响重置。

2. **⚠️ JSON 格式大坑**：删完字段后，**它上一行的字段结尾很可能还留着一个逗号 `,`**——这个多余的尾逗号会让整个 JSON 非法，仪表盘启动直接报错。删字段时必须把那个尾逗号一起删掉（让上一行以 `}` 正常结尾）。

   > 好消息：从本版起，仪表盘启动时已内置**自动修复**——就算你手滑留了尾逗号，它也会自动修好再读，不会崩溃。但稳妥起见还是顺一眼括号 `{ }` 是否配对。

3. 存盘，重新运行 `python engine/dashboard.py`。框架发现没有密码字段了，会当作首次启动重新生成初始密码并显示在登录页——用它登录，再按需重新设新密码。

### 这样安全吗？为什么不做"网页一键重置"

安全，而且这是**刻意的设计**：

- 能删这个文件 = 你能物理访问这台机器、能读写它的硬盘。对一个**跑在你自己电脑上的本地工具**来说，这就是它的信任边界——不是网上随便谁都能重置。
- 框架**故意不提供"网页上一键重置密码"的按钮**。那种不需要登录就能点的重置入口，本质是个后门：任何能打开这个网页的人都能一键重置、直接进来看你的记忆。把重置权收敛到"能碰到你硬盘上的文件"，才真正安全。

> **重要**：删这三行**只影响仪表盘登录密码**，你的记忆数据（`data/` 文件夹里的一切）**完全不受影响**，一条都不会丢。

## 外挂会不会搞乱我原本用的 AI？

**不会。** 这是框架的硬原则（详见 `rules/host-isolation.md`）：

- 框架**只写自己的本子**（`data/`），**绝不碰** OpenClaw / WorkBuddy 等宿主各自的记忆文件、配置、日志。
- 框架是**平行叠加层，不是覆盖层**——它不凌驾于宿主记忆之上。
- 你的原 AI 该怎么记还怎么记，框架只是在对话里额外带一段"它懂你什么"的上下文。

## 你的数据在哪 / 隐私说明

- 所有"它了解你的内容"都存在你机器上的 `data/facts/*.jsonl`，**纯文本**，你能随时打开看。
- 想用图形化界面看？跑 `python engine/dashboard.py`，浏览器打开所有数据，搜索/筛选/分页一应俱全。
- 框架**出厂不含任何人的数据**，全是你们聊出来的。
- 框架**不会把你的数据上传任何地方**（除非你自己改代码去接云服务）。
- 想"重置关系"：清空 `data/facts/` 文件夹即可，配置不动。
- 想"让它忘掉某件事"：删掉 `data/facts/` 里对应那一行，或让它"忘掉 XXX"。

## 常见问题

**Q：我用的是别人做好的 AI，改不了代码，怎么让它"自动"带上记忆？**
如果你的 AI 支持 MCP（2026 年主流都支持，包括 WorkBuddy、OpenClaw 这类"别人做好的 AI"），直接挂 MCP（B 路）——它自己读自己写，全自动，你不用粘任何东西。
只有确认它不支持 MCP，才用 **A 粘贴法**兜底：每次开聊先把 `make_context.py` 生成的文本粘进对话框。
> 说明：我之前文档里写"闭源网页 AI 用不了 MCP"是**错的**——WorkBuddy 本身就有 MCP 配置入口，连它都能接。已更正。

**Q：我换了 AI（比如从这个模型切到另一个），它还懂我吗？**
懂。因为记忆在框架里、不在模型里。只要新 AI 接上同一个框架文件夹（或你在新 AI 里粘贴同一份 context），它就带着旧记忆上岗。

**Q：它会乱记、乱分析我吗？**
四条底线锁死，任何配置都改不了：
不操控你 · 不越界揣测 · 不伪装真人 · 不降格对待。详见 `rules/boundaries.md`。

**Q：我说话很随意，它能读懂吗？**
能。框架内置"主动分析"机制（按你的语气、措辞、行为推断状态），而且是**默默分析、不当面给你贴标签**。

**Q：需要联网吗？**
不需要。框架纯本地运行。

**Q：配置填错了怎么办？**
改 `config.schema.json` 重新填就行，不影响已记的内容。或者更简单——接上 MCP 后直接对话里让 AI 改（"你叫 XX""我是老李"），它自己经 `update_config` 写回。

**Q：AI 能自己改昵称/我的称呼吗？**
能。挂上 MCP 后，你说"你叫 XX"或"以后叫我老李"，AI 调 `update_config` 自己写进配置，下次对话立刻用新名字。但它只能改昵称/称呼/关系/基调这四个，改不了框架内部路径（防改坏）。

**Q：用几年后，记忆文件会不会太大、撑爆每次对话的 token？**
不会失控。框架有四层护城河：① 每次 recall 有**字符预算**（默认 1800），超出按"重要性+最近性"自动截断，token 成本恒定；② Vault 走廊预载（最近 N 条 active）让 AI 快速感知近期上下文，无需翻全量 JSONL；③ `compact_memory` 定期把旧记录**蒸馏归档**到 `cold/` + 触发 vault 滚动（走廊→房间），热层只留近期高价值内容；④ 深挖旧事时 AI 调 `vault_search` 或 `get_fact` 才穿透冷层捞回。磁盘上纯文本几年也就 MB 级，根本不是瓶颈——真正被管住的是 token。

**Q：多个 AI（多智能体）能共用一个灵魂盘吗？**
能。靠 `agent_id`（翼标识 wing）分区隔离：
- **公共区**（`wing="cross"`）：FACT/知识/经验 → 全员可读写互学
- **私密区**（`wing=各自agent_id`）：PREF/BOUND/COMMIT → 各智能体独立，互不可见
- 搜索时不传 wing = 搜全员（互相学习）；预载只拉自己的走廊（不窥探）
- 这就是"一个灵魂盘，多个智能体，各自有隐私，又能共享知识"的设计。

**Q：我怎么亲手看看灵魂盘里都记了什么？**
跑 `python engine/dashboard.py`，浏览器打开实时仪表盘——所有记忆分标签页展示，搜索/排序/分页，一目了然。也支持导出静态 HTML：`python engine/dashboard.py --export`。

**Q：仪表盘管理员密码忘了怎么办？**
最简单：关掉仪表盘，在框架目录运行 `python engine/dashboard.py --reset-admin`，它会直接重置并打出新初始密码，不动任何记忆数据。手动删字段法见上文《仪表盘登录 & 忘记管理员密码怎么办》。

**Q：源码更新了，怎么升级？会不会丢记忆数据？**
**不会丢。** 源码是"壳"（零数据），`data/` 文件夹才是你的记忆。升级三步：
1. 从 GitHub 下载最新源码，解压到一个**新文件夹**。
2. 把你**旧文件夹**里的 `data/` 整个目录，复制到新文件夹里（新源码出厂 `data/` 是空的，不丢任何东西）。
3. **`config.schema.json` 不要覆盖**——旧实里是你配好的真配置（名称/关系/密钥等），新源码里是空模板，覆盖等于白手起家。保留旧的就行，新字段代码会自动用默认值兜底。
4. 新文件夹就是升级后的实例——跑一下试试，确认正常后旧文件夹可以删。

> 一句话：**源码只管覆盖，`data/` 搬过去，`config.schema.json` 不碰——记忆一条不少，配置一个不缺。**
